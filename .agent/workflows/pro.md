# 🧠 全域产品经理工作流 (PM-Mode SOP)

当用户输入 `Solve-PM:`、`PM-mode:` 或涉及“统筹多个功能/不断补充知识”的需求时，**必须**执行以下标准化作业程序：

## 1. 宪法级需求解构 (Governance & Spec-Driven)
- **[CRITICAL] 时间锚点对齐**：必须确认当前年份（2026）。搜寻时强制带 `latest` 或 `2026` 标签。
- **引用核心协议**：读取 `d:/anti/技能库/05_极简审美与动画/core/pm_governance/GOD_PM_CORE_2026.md`（原 Skill 00）。
- **递归反映规划 (Reflection)**：在生成 `implementation_plan.md` 前，模拟 2 个以上专家视角进行自我批判。
- **置信度自审**：评分必须 > 9.5 才可进入下一步。

## 2. 存量资产审计 (Asset Audit)
- **技能扫描**：读取 `SKILL_INDEX.md` 和 `技能库/` 下所有 `SKILL.md` 或 `README.md`。
- **项目对齐**：检查 `PROJECT_INDEX.md`，确认是否有重复或冲突的逻辑。

## 3. 深度研究 (Phase 1: Deep Search - Skill 01)
- **必选动作**：执行 `search_web` 获取行业最佳实践内容。
- **搜寻三位一体**：执行 `Skill 06` 中的 GitHub 与社区扫描逻辑。
- **知识内化**：将搜集到的技术点总结并写入临时研究报告。

## 4. 架构设计与决策 (Phase 2: Arch Design - Skill 06)
- **方案对比**：制作评估矩阵（成本、维护性、扩展性）。
- **终极方案建议**：给出单一明确的 Recommended 路径。

## 5. 项目记忆固化 (Knowledge Persistence)
- **建立项目文件夹**：`d:/anti/projects/[TaskName]/`。
- **生成 PRD.md**：记录需求、选型建议与未来扩展的方向。
- **补充知识库**：如果学到了新技能（如新 API 用法），更新对应的 `技能库` 文件。

## 6. 巅峰交付 (Professional Delivery)
- **一键触达 (HTTP Server)**：交付所有网页/UI 设计时，必须启动本地服务器。
- **演练文档 (Walkthrough)**：包含实机截图，建立知识闭环。
